# How A.I tools were used

## What was used

The primary tool used was Gemini, via the website.
I did not have much time to setup and install more code focuses A.I tools such as claude, copilot or gemini cli.
Also because some of the A.I tools wanted to gain full write access to all my github repositories, which is a huge
security flaw.

## How it was used

I primarily used Gemini like a second opinion for any coding issues I may have.
With Gemini, it can output good ideas sometimes, although it takes a bit of prompt engineering to do so.
You have to be quite specific and clear in your goals/questions and it can suggest pretty good ideas.

However with regards to code itself, it is still not the best and sometimes the occasional hallucination happens.
Sometimes, it can completely miss essentials parts of code that was previously generated.
Also, Gemini context window is not that large, it can forget quite quickly what was discussed a few days ago.

The various patterns used in the code, i.e, Visitor pattern and Iterator pattern (stream) was partially suggested by
Gemini.
I took inspirations from its generated code and wrote the code myself.

I also used Gemini to generate some of the javadocs, although I realised it was getting to verbose,
even after asking it to summarise multiple times.

Overall, I would say that it did not save as much time as I thought, sometimes taking more time because it does not
generate what I expected it to. Thus, more time was spent to correct it.
